# Security Policy

## Supported Versions

The repository is maintained on the `main` branch. Security fixes are applied
to the current public branch rather than backported to historical snapshots.

## Reporting a Vulnerability

Please do not open a public GitHub issue for suspected vulnerabilities.

Report security issues privately by email to:

- `a.belilovsky@gmail.com`

Include:

- a short description of the issue
- affected files, endpoints, or flows
- reproduction steps or proof of concept
- impact assessment if known

We aim to acknowledge serious reports promptly and coordinate a fix before
public disclosure.
