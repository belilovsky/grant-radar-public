# Экспертная проверка ранжирования QAZ.FUND

Дата введения: 2026-07-16  
Формат очереди: `qazfund-ranking-review-v1`

## Зачем нужен отдельный контур

Unit-тесты проверяют формулу и граничные случаи, но не доказывают, что порядок
реальных возможностей полезен редактору, аналитику или заявителю. Этот контур
отделяет три разных вида доказательств:

1. детерминированные policy-тесты модели;
2. экспертную оценку реальных карточек;
3. будущие исходы подачи заявок, которые нельзя подменять экспертной оценкой.

Результат не является вероятностью одобрения заявки и не заменяет юридическую
проверку критериев программы.

## Подготовка очереди

```bash
make ranking-review-sample
```

Команда читает публичный API и создает локальный JSONL-файл из 200 реальных
карточек. Выборка детерминированно чередует источники, географию, уровень
уверенности и диапазоны релевантности. Рабочий файл находится в
`output/ranking-evaluation/` и не коммитится: он может содержать имена
рецензентов и редакционные заметки.

Повторная генерация требует явного `--force`, поэтому начатая разметка не
перезаписывается случайно.

## Шкалы

### Релевантность каталога

| Оценка | Значение |
| ---: | --- |
| `0` | Не относится к аудитории QAZ.FUND или имеет явное исключение |
| `1` | Слабая или неподтвержденная применимость, нужна дополнительная проверка |
| `2` | Практически применимо хотя бы для одного целевого сегмента |
| `3` | Прямое и хорошо подтвержденное совпадение для Казахстана или Центральной Азии |

### Приоритет действия

| Оценка | Значение |
| ---: | --- |
| `0` | Действовать нельзя или не нужно: закрыто, нерелевантно, недостаточно данных |
| `1` | Наблюдать или сначала подтвердить существенные условия |
| `2` | Можно брать в план и начинать проверку/подготовку |
| `3` | Стоит открыть и квалифицировать сейчас, срок и применимость это позволяют |

Каждая оцененная строка обязана содержать `reviewer` и `reviewed_at` с часовым
поясом. Схема опубликована в
[`schemas/ranking-review-v1.schema.json`](schemas/ranking-review-v1.schema.json).

## Процесс согласования

1. Два редактора независимо оценивают не менее 20% выборки.
2. Расхождение на два и более балла обязательно разбирается совместно.
3. Главный редактор принимает согласованную оценку, не меняя model output.
4. Карточки с юридически неоднозначной применимостью получают редакционную
   заметку, но не искусственно повышенный балл.
5. После изменения модели создается новая очередь: версии нельзя смешивать в
   одном отчете.

## Расчет отчета

```bash
make ranking-evaluate
```

Отчет включает отдельно для релевантности и приоритета:

- `Precision@10`;
- `Recall@50`;
- `nDCG@10` для градуированных оценок `0-3`;
- число и долю уникальных источников в первой десятке;
- среднюю абсолютную ошибку относительно нормированной экспертной шкалы;
- доли ложноположительных и ложноотрицательных решений на публичном пороге;
- покрытие по источникам, географии, формату, теме и уровню уверенности.

Evaluator завершится ошибкой, если отсутствует разметка, встречаются дубли,
смешаны версии модели, нарушена шкала или не указано авторство оценки.

## Начальный release gate

До принятия первого согласованного gold set пороги считаются предварительными и
не включаются в CI. CLI по умолчанию требует:

- не менее 200 оценок для обоих показателей;
- relevance `Precision@10 >= 0.75`, `nDCG@10 >= 0.75`;
- relevance false-positive rate `<= 0.20` на пороге `0.30`;
- action priority `Precision@10 >= 0.70`, `nDCG@10 >= 0.70`.

После первого согласования пороги фиксируются относительно принятого baseline.
Ослабление gate допускается только отдельным документированным решением, а не
для прохождения конкретного релиза.

## Проверенный dry run

Dry run на production API 2026-07-16:

- исходный каталог: 468 карточек;
- очередь: 200 уникальных карточек;
- источники: 24;
- география: 97 Казахстан, 83 Центральная Азия, 20 глобальных;
- confidence: 180 `supported`, 20 `review_required`;
- добавленные автоматически экспертные оценки: 0;
- gate без разметки: ожидаемо `failed`.

Это проверка работоспособности sampling/evaluation pipeline, а не оценка качества
самой модели.
